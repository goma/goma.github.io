Welcome to the new BETA Quick-Build page for Goma 6.0! We have made a run script that will allow you to easily download all of the necesarry supporting files for Goma 6.0, and we have also included simple instructions on how to correctly configure Goma 6.0 so that it is ready for immediate use!

This script has been tested on CentOS 6, CentOS 7, and Redhat 6

Download the Goma Library Build script

Download the Goma Library build script here: Goma Library Script

Use the script in the following way: bash build-goma-dependencies.sh -j[Number of Make Jobs] [Build/Install Location]

As soon as that finishes downloading, you should have the essential libraries inorder to run Goma 6.0!

Before building goma you should add the newly compiled openmpi, and SEACAS to your path

If you want easy access you can add these settings to your shell settings (e.g. .bashrc) otherwise you can type them in manually each time you want to run goma

OPENMPI:

  export LD_LIBRARY_PATH="/[path to gomalibs]/openmpi-1.6.4/lib:$LD_LIBRARY_PATH"
  export PATH="/[path to gomalibs]/openmpi-1.6.4/bin:$PATH"

SEACAS

  export PATH="/[path to gomalibs]/SEACAS-2013-12-03/bin:$PATH"

Download Goma and Brkfix from github: Goma on github

To build goma copy the settings.mk-example to settings.mk set the GOMA_LIBS macro to match your specified installation directory and add the makefile macro FRONT_LIB = to your settings.mk file.

Goma should now be able to compile with make

To build brkfix adjust the ACCESS to match your seacas install from the build script [Install Location]/SEACAS-2013-12-03
